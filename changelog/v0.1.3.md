# v0.1.3 — 2026-04-24

Sidebar UX and tab persistence.

## Added

- **Restore tabs across restarts.** Open tabs reopen dormant — the tab title is preserved but `claude` is not spawned until you click **Resume** inside the tab (or **Start** for new-kind tabs).
- **Restore expanded sidebar folders.** Which project folders you had open is remembered between launches.
- **Session count per folder.** Each project row now shows a `N sessions` caption.
- **Per-project filesystem watcher.** New sessions started inside the app appear in the sidebar without waiting for a full rescan. Claude's initial summary shows up once it's written.

## Changed

- **Sidebar folder rows redesigned** — folder icon, project name, and session count, with padding that matches the session rows below.
- **Auto-refresh preserves expansion state** — opening a new session no longer collapses the folder you were in.

## CI

- Added a `publish-aur` job to the `Release` workflow that pushes `packaging/aur/PKGBUILD` (with regenerated `.SRCINFO`) to the AUR repo on every tag release. Requires the `AUR_SSH_PRIVATE_KEY` repository secret.

## Config schema

Two new persisted fields in `~/.config/persistent-claude-code/config.json`:

| Key | Purpose |
| --- | --- |
| `open_tabs` | List of `{kind, session_id, title, cwd}` entries — the tabs to restore on next launch. |
| `expanded_projects` | List of project cwd paths whose sidebar folder was expanded. |

Both are updated on window close and safe to edit or delete manually.
