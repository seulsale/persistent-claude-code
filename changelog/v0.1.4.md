# v0.1.4 — 2026-04-28

Terminal clipboard wiring.

## Fixed

- **`Ctrl+Shift+V` now pastes** inside the terminal. Previously the keystroke was passed through to the child process, which (because terminals encode `Ctrl+V` and `Ctrl+Shift+V` as the same byte) made Claude Code interpret it as `Ctrl+V` and reply with "no image in clipboard."
- **Right-click in the terminal** now opens a small **Copy / Paste** popover. Both actions also work from the keyboard.

## Added

- `Ctrl+Shift+C` copies the current terminal selection to the clipboard.

## Notes

VTE in GTK4 ships without default copy/paste keybindings or a context menu — embedding apps have to wire them up. We now register a `Gtk.ShortcutController` on the VTE widget for the keyboard shortcuts and a right-click `Gtk.PopoverMenu` for the mouse path.
